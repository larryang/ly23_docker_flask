Python Development

Information on developing for python. Description and usage of various tools to enforce quality and testing. The tools listed in this page help to ensure quality of the code, and implement concepts and processes for Python software development (such as CI/CD and AAA testing).

  • pylint - static analysis tool
  • pep8 - style guide for Python Code
  • pytest - Python test framework

pylint

pylint is a static analysis tool, i.e. it checks the source code for without executing the code. It's looking for common programming errors, suspicious code, etc. It is executed everytime as part of the CI/CD pipeline, and any issues found will cause the flagged as an error that should be corrected. It is also enabled in the Visual Code editor, and executed everytime the file is saved.

pep8

pep8 is the official Pyhton style guide. It makes the code style consistent, easier to read and maintain by humans. Things like alignment, variable naming, spacing, comment style, line length are enforced. It is enforced through pylint in CI/CD and also enabled in the Visual Code editor.

pytest

pytest is the Python test framework. It implements the Arange, Act, Assert Testing, and is executed everytime in the CI/CD pipeline. More detail on how pytest is used can be found on the AAA Testing information page of this website.
